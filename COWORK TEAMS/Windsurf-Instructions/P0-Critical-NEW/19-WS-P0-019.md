# WS-P0-019: Immutable Document Hash-Chain

**Priority:** P0-CRITICAL
**Team:** Alpha (Backend & Data)
**Estimated Complexity:** High
**Unlocks:** Document trust, tamper detection, audit trail

## Problem
The current signature system in `signatures.ts` is **cosmetic, not cryptographic**. It generates a "SHA256" string from the first 100 chars of signature data encoded as base64, but this is NOT a real content hash. The verification always returns `isValid: true` without checking anything. There is no hash-chain linking document versions. Sales materials promise "immutable documentation" to shippers and brokers.

## What to Implement

### 1. New schema table: `documentHashes`
```sql
CREATE TABLE document_hashes (
  id SERIAL PRIMARY KEY,
  documentId INT NOT NULL,
  documentType VARCHAR(50) NOT NULL, -- 'bol', 'pod', 'agreement', 'invoice', 'settlement'
  version INT NOT NULL DEFAULT 1,
  contentHash VARCHAR(64) NOT NULL, -- SHA256 hex digest
  previousHash VARCHAR(64), -- hash of previous version (null for v1)
  chainHash VARCHAR(64) NOT NULL, -- SHA256(contentHash + previousHash)
  createdBy INT NOT NULL,
  createdAt TIMESTAMP DEFAULT NOW(),
  metadata JSONB,
  UNIQUE(documentId, documentType, version)
);
```

### 2. Hash service: `frontend/server/services/documents/hashChain.ts`
```typescript
import crypto from 'crypto';

export function hashContent(content: Buffer | string): string {
  return crypto.createHash('sha256').update(content).digest('hex');
}

export function computeChainHash(contentHash: string, previousHash: string | null): string {
  const input = previousHash ? `${contentHash}:${previousHash}` : contentHash;
  return crypto.createHash('sha256').update(input).digest('hex');
}

export async function addToChain(
  documentId: number,
  documentType: string,
  content: Buffer | string,
  userId: number,
) {
  const contentHash = hashContent(content);

  // Get previous version
  const prev = await db.select()
    .from(documentHashes)
    .where(and(
      eq(documentHashes.documentId, documentId),
      eq(documentHashes.documentType, documentType),
    ))
    .orderBy(desc(documentHashes.version))
    .limit(1);

  const previousHash = prev.length ? prev[0].chainHash : null;
  const version = prev.length ? prev[0].version + 1 : 1;
  const chainHash = computeChainHash(contentHash, previousHash);

  await db.insert(documentHashes).values({
    documentId, documentType, version,
    contentHash, previousHash, chainHash,
    createdBy: userId,
  });

  return { contentHash, chainHash, version };
}

export async function verifyChain(
  documentId: number,
  documentType: string,
): Promise<{ valid: boolean; brokenAt?: number }> {
  const chain = await db.select()
    .from(documentHashes)
    .where(and(
      eq(documentHashes.documentId, documentId),
      eq(documentHashes.documentType, documentType),
    ))
    .orderBy(asc(documentHashes.version));

  for (let i = 0; i < chain.length; i++) {
    const expected = computeChainHash(
      chain[i].contentHash,
      i > 0 ? chain[i - 1].chainHash : null
    );
    if (expected !== chain[i].chainHash) {
      return { valid: false, brokenAt: chain[i].version };
    }
  }
  return { valid: true };
}
```

### 3. Wire into document creation flows
- BOL creation/signing in loads.ts

### 4. Verification API endpoint
```typescript
// In a public or protected router
verifyDocument: publicProcedure
  .input(z.object({ documentId: z.number(), documentType: z.string() }))
  .query(async ({ input }) => {
    return verifyChain(input.documentId, input.documentType);
  }),
```

## Acceptance Criteria
- [ ] Every BOL created gets a SHA256 content hash stored in documentHashes
- [ ] Every POD submitted gets hashed and chained to the BOL hash
- [ ] `verifyChain()` returns `{ valid: true }` for untampered documents
- [ ] `verifyChain()` returns `{ valid: false, brokenAt: N }` when chain is broken
- [ ] API endpoint `/api/documents/:id/verify` returns chain validity
- [ ] Old cosmetic "SHA256:base64..." removed from signatures.ts

## Customer Impact
Shippers and Brokers were explicitly promised "immutable documentation with hash-chain verification."
