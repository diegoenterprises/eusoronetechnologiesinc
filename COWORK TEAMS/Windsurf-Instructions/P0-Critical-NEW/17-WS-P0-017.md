# WS-P0-017: Compliance Gate Before Load Assignment

**Priority:** P0-CRITICAL
**Team:** Delta (Compliance & Regulatory)
**Estimated Complexity:** Medium
**Unlocks:** Shipper trust, FMCSA regulatory compliance

## Problem
The `accept()` mutation in `loadBidding.ts` (lines 318-349) assigns loads to carriers with ZERO compliance verification. No FMCSA safety rating check, no operating authority validation, no insurance verification. A carrier with an "Unsatisfactory" FMCSA rating or expired authority can be assigned hazmat crude oil loads.

## What to Implement

Add a compliance verification step BEFORE the load assignment in the `accept()` mutation:

```typescript
// In loadBidding.ts accept() mutation, BEFORE updating load status:

// 1. Get the bidder's company
const bidderCompany = await db.select()
  .from(companies)
  .where(eq(companies.userId, bid.bidderUserId))
  .limit(1);

if (!bidderCompany.length || !bidderCompany[0].dotNumber) {
  throw new TRPCError({
    code: 'FORBIDDEN',
    message: 'Carrier must have a registered DOT number to accept loads',
  });
}

// 2. Check FMCSA safety rating
const fmcsaData = await getFMCSASafetyData(bidderCompany[0].dotNumber);
if (fmcsaData.safetyRating === 'Unsatisfactory') {
  throw new TRPCError({
    code: 'FORBIDDEN',
    message: 'Carrier has Unsatisfactory FMCSA safety rating',
  });
}

// 3. Verify operating authority is active
if (fmcsaData.operatingAuthority?.status !== 'Active') {
  throw new TRPCError({
    code: 'FORBIDDEN',
    message: 'Carrier operating authority is not active',
  });
}

// 4. Verify insurance meets load requirements
const insurance = await db.select()
  .from(insuranceCertificates)
  .where(and(
    eq(insuranceCertificates.companyId, bidderCompany[0].id),
    eq(insuranceCertificates.status, 'active'),
    gte(insuranceCertificates.expirationDate, new Date()),
  ));

if (!insurance.length) {
  throw new TRPCError({
    code: 'FORBIDDEN',
    message: 'Carrier has no valid insurance certificates',
  });
}

// 5. Log the compliance decision
await db.insert(complianceAuditLog).values({
  action: 'load_assignment_compliance_check',
  carrierId: bid.bidderUserId,
  loadId: bid.loadId,
  result: 'passed',
  details: JSON.stringify({
    fmcsaRating: fmcsaData.safetyRating,
    authorityStatus: fmcsaData.operatingAuthority?.status,
    insuranceCount: insurance.length,
  }),
});

// THEN proceed with existing assignment logic...
```
