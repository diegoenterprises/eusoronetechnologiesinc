## ISSUE

Wallets with userId=0 and userId=2 reference non-existent users. These orphaned records will cause payment processing failures.

## REQUIRED CHANGE

```sql
-- Delete orphaned wallet records
DELETE FROM wallets WHERE userId NOT IN (SELECT id FROM users);

-- Add foreign key constraint to prevent future orphans (if not already present)
ALTER TABLE wallets ADD CONSTRAINT fk_wallets_userId
  FOREIGN KEY (userId) REFERENCES users(id) ON DELETE CASCADE;
```

If the FK constraint fails due to existing violations, clean up first.

## VERIFICATION

```sql
SELECT COUNT(*) FROM wallets WHERE userId NOT IN (SELECT id FROM users);
-- Should return 0
```
