## ISSUE

`loadLifecycle.ts` transitions load states in the database but NEVER calls `emitLoadStatusChange()`. The WebSocket infrastructure exists (140+ event types defined, native WS server running) but load lifecycle events are never emitted. Dispatch boards, shipper dashboards, and driver apps must poll for updates instead of receiving real-time push.

The import for `emitLoadStatusChange` already exists at the top of `loads.ts` — it's just never called after state transitions.

## AFFECTED USER TYPES

ALL — Shipper (tracking), Catalyst (dispatch board), Driver (status), Terminal Manager (incoming), Broker (monitoring)

## AFFECTED WORKFLOW

Load state transition → WebSocket broadcast → All connected clients update in real-time

## FILE(S) TO MODIFY

`frontend/server/routers/loadLifecycle.ts` — after each `transitionState()` DB update

Reference: `frontend/server/_core/websocket.ts` — contains `emitLoadStatusChange`, `emitNotification`

## REQUIRED CHANGE

In `loadLifecycle.ts`, after every successful state transition, add WebSocket emission:

```typescript
// After successful DB state update in transitionState():
import { emitLoadStatusChange, emitNotification } from '../_core/websocket';

// Inside the transition handler, after DB update succeeds:
emitLoadStatusChange({
  loadId: input.loadId,
  loadNumber: load.loadNumber,
  fromState: currentState,
  toState: input.newState,
  triggeredBy: ctx.user.id,
  triggeredByRole: ctx.user.role,
  timestamp: new Date().toISOString(),
  // Include location if GPS-triggered
  location: input.location || null,
});

// Also emit role-specific notifications:
// Notify shipper when their load status changes
if (load.shipperId) {
  emitNotification({
    userId: load.shipperId,
    type: 'load_status_change',
    title: `Load ${load.loadNumber} is now ${input.newState}`,
    body: `Your load has transitioned from ${currentState} to ${input.newState}`,
    data: { loadId: input.loadId, newState: input.newState },
  });
}

// Notify catalyst when assigned load changes
if (load.catalystId) {
  emitNotification({
    userId: load.catalystId,
    type: 'load_status_change',
    title: `Load ${load.loadNumber}: ${input.newState}`,
    data: { loadId: input.loadId, newState: input.newState },
  });
}

// Notify driver
if (load.driverId) {
  emitNotification({
    userId: load.driverId,
    type: 'load_status_change',
    title: `Load ${load.loadNumber}: ${input.newState}`,
    data: { loadId: input.loadId, newState: input.newState },
  });
}
```

This should be added to ALL state transitions, including:
- posted → bidding
- bidding → awarded
- awarded → assigned
- assigned → dispatched
- dispatched → en_route_pickup
- at_pickup → loading
- loading → in_transit
- in_transit → at_delivery
- at_delivery → delivered
- delivered → paid

Also add for geofence-triggered transitions (if they exist in the code).
