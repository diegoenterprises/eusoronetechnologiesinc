## ISSUE

`wallet.requestPayout()` in `wallet.ts` creates a database record but NEVER actually calls `stripe.transfers.create()` or `stripe.payouts.create()`. Drivers and catalysts see earnings in their wallet but cannot withdraw money. The funds are trapped.

## AFFECTED USER TYPES

Catalyst/Carrier (primary), Driver, Escort, Factoring, Broker

## AFFECTED WORKFLOW

Catalyst/Driver Payout: Wallet → Request Payout → Bank Account

## FILE(S) TO MODIFY

`frontend/server/routers/wallet.ts` — the `requestPayout` procedure (around line 500+)

## REQUIRED CHANGE

In the `requestPayout` mutation, after the database record is created, add actual Stripe transfer execution:

```typescript
// After creating the DB record in wallet_transactions...

// 1. Get user's Stripe Connect ID
const { stripeConnectId } = await ensureStripeCustomer(db, ctx.user.id);

if (!stripeConnectId) {
  throw new TRPCError({
    code: 'PRECONDITION_FAILED',
    message: 'Please connect your bank account via Stripe Connect before requesting a payout.',
  });
}

// 2. Execute the Stripe transfer with idempotency
const transfer = await safeStripeCall(() =>
  stripe.transfers.create({
    amount: Math.round(input.amount * 100), // cents
    currency: 'usd',
    destination: stripeConnectId,
    description: `EusoTrip payout for ${ctx.user.name || ctx.user.email}`,
    metadata: {
      userId: String(ctx.user.id),
      withdrawalId: `wd_${Date.now()}`,
      platform: 'eusotrip',
    },
  }, {
    idempotencyKey: `payout_${ctx.user.id}_${Date.now()}`,
  })
);

// 3. Update the wallet_transaction record with Stripe transfer ID
if (transfer?.id) {
  await db.update(walletTransactions)
    .set({
      stripeTransferId: transfer.id,
      status: 'processing',
    })
    .where(eq(walletTransactions.id, txnId));

  // 4. Debit the wallet balance
  await db.update(wallets)
    .set({
      availableBalance: sql`${wallets.availableBalance} - ${input.amount}`,
      pendingBalance: sql`${wallets.pendingBalance} + ${input.amount}`,
    })
    .where(eq(wallets.userId, ctx.user.id));
} else {
  // Stripe call failed — mark as failed, don't debit wallet
  await db.update(walletTransactions)
    .set({ status: 'failed', notes: 'Stripe transfer failed' })
    .where(eq(walletTransactions.id, txnId));

  throw new TRPCError({
    code: 'INTERNAL_SERVER_ERROR',
    message: 'Payout failed. Please try again or contact support.',
  });
}
```

Also: ensure the `walletTransactions` schema has a `stripeTransferId` column (varchar, nullable). If missing, add a database migration.
