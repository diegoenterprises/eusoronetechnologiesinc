## ISSUE
Add secondary LLM fallback for ESANG. If Gemini API is down, ESANG AI completely fails. No fallback provider for continuity.

## AFFECTED USER TYPES
- All users who use ESANG

## FILE(S) TO MODIFY
- frontend/server/_core/esangAI.ts
- .env (add fallback API keys)

## REQUIRED CHANGE

**.env additions:**
```
GEMINI_API_KEY=xxx
OPENAI_API_KEY=yyy
ANTHROPIC_API_KEY=zzz
```

**esangAI.ts fallback pattern:**
```typescript
async function esangQuery(prompt: string): Promise<string> {
  const systemPrompt = 'You are ESANG, expert shipping agent...';

  // Try Gemini first
  try {
    const response = await gemini.generateContent({
      contents: [{ role: 'user', parts: [{ text: prompt }] }],
      systemInstruction: systemPrompt,
    });
    return response.response.text();
  } catch (geminiErr) {
    console.warn('[ESANG] Gemini failed:', geminiErr.message);
  }

  // Fallback to OpenAI
  try {
    const response = await openai.chat.completions.create({
      model: 'gpt-4o-mini',
      system: systemPrompt,
      messages: [{ role: 'user', content: prompt }],
    });
    return response.choices[0].message.content;
  } catch (openaiErr) {
    console.warn('[ESANG] OpenAI failed:', openaiErr.message);
  }

  // Final fallback to Anthropic Claude
  try {
    const response = await anthropic.messages.create({
      model: 'claude-3-5-sonnet-20241022',
      system: systemPrompt,
      messages: [{ role: 'user', content: prompt }],
    });
    return response.content[0].text;
  } catch (claudeErr) {
    console.error('[ESANG] All LLM providers failed');
    throw new Error('ESANG unavailable');
  }
}
```

## VERIFICATION
1. Set invalid Gemini key
2. Query ESANG → falls back to OpenAI/Claude
3. Set all keys invalid → proper error
4. Restore Gemini key → uses Gemini first (preferred)
5. Verify response quality consistent across providers
