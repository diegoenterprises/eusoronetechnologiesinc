## ISSUE
Integrate hash-chain for compliance audit immutability. hash-chain.ts exists with SHA-256 chaining but isn't wired into compliance decisions. Audit defense is weaker without tamper-evident chain.

## AFFECTED USER TYPES
- Compliance Officer
- Admin

## FILE(S) TO MODIFY
- frontend/server/services/security/audit/hash-chain.ts (wire into compliance gate)
- compliance.ts (compliance gate from WS-P0-005)

## REQUIRED CHANGE

**In compliance gate (compliance.ts):**
```typescript
import { HashChain } from '../services/security/audit/hash-chain';

const hashChain = new HashChain('compliance-audit');

async function complianceGateDecision(
  driverId: string,
  gateType: string,
  passed: boolean
) {
  const decision = {
    driverId,
    gateType,
    passed,
    timestamp: new Date().toISOString(),
    metadata: { checkVersion: '2.1' },
  };

  // Create hash-chain entry linking to previous
  const entry = hashChain.append(decision);

  await db.query(
    `INSERT INTO compliance_audit_chain
     (chain_entry_id, driver_id, gate_type, passed, entry_hash, prev_hash)
     VALUES ($1, $2, $3, $4, $5, $6)`,
    [entry.id, driverId, gateType, passed, entry.hash, entry.prevHash]
  );

  return entry;
}

// Verify chain integrity
async function verifyChainIntegrity() {
  const entries = await db.query(
    `SELECT * FROM compliance_audit_chain ORDER BY created_at`
  );

  let currentHash = null;
  for (const entry of entries) {
    if (entry.prevHash !== currentHash) {
      throw new Error('Chain tampered at entry: ' + entry.id);
    }
    currentHash = entry.entryHash;
  }
  return true;
}
```

## VERIFICATION
1. Run compliance gate check → entry added to hash-chain
2. Query audit entries → continuous chain visible
3. Manually alter audit entry in DB
4. Run verifyChainIntegrity() → detects tampering
