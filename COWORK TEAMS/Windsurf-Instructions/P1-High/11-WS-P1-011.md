## ISSUE
Validate FMCSA API key on startup. If FMCSA_API_KEY is missing or invalid, all carrier safety lookups silently fail with no startup health check.

## AFFECTED USER TYPES
- Compliance Officer
- Admin

## FILE(S) TO MODIFY
- Server startup file (e.g., frontend/server/index.ts or server.ts)
- .env file (add validation)

## REQUIRED CHANGE

**In server startup (frontend/server/index.ts):**
```typescript
async function validateFmcsaApiKey() {
  const apiKey = process.env.FMCSA_API_KEY;

  if (!apiKey) {
    console.warn(
      '[STARTUP] WARNING: FMCSA_API_KEY not set. ' +
      'Carrier safety lookups will fail.'
    );
    return false;
  }

  try {
    // Test with known DOT number
    const response = await fetch(
      `https://api.fmcsa.dot.gov/public/inspection/query?dotNumber=1234567`,
      { headers: { 'X-API-Key': apiKey } }
    );
    if (!response.ok) throw new Error('API returned non-200');
    console.log('[STARTUP] FMCSA API key validated successfully');
    return true;
  } catch (err) {
    console.warn('[STARTUP] WARNING: FMCSA API key validation failed:', err.message);
    return false;
  }
}

app.listen(PORT, async () => {
  await validateFmcsaApiKey();
  console.log(`Server running on port ${PORT}`);
});
```

## VERIFICATION
1. Remove FMCSA_API_KEY from .env
2. Restart server → see warning in startup logs
3. Restore API key
4. Restart server → validation succeeds
5. Verify compliance officers notified of key validity
