## ISSUE
Add unique constraint on load_bids. No unique constraint on load_bids(loadId, bidderUserId) allows duplicate bids from same user on same load, creating data integrity violations.

## AFFECTED USER TYPES
- Catalyst
- Shipper

## FILE(S) TO MODIFY
- Database migration file (create new: migrations/YYYYMMDD_add_unique_bid_constraint.sql)

## REQUIRED CHANGE

**Create migration:**
```sql
-- Migration: Add unique constraint on load_bids
ALTER TABLE load_bids
ADD CONSTRAINT idx_unique_bid UNIQUE (load_id, bidder_user_id);

-- Index for performance
CREATE UNIQUE INDEX idx_unique_bid_lookup
ON load_bids(load_id, bidder_user_id);
```

**Application-level validation (frontend/server/routers/bids.ts):**
```typescript
async function submitBid(loadId: number, userId: string, amount: number) {
  const existing = await db.query(
    `SELECT id FROM load_bids WHERE load_id = $1 AND bidder_user_id = $2`,
    [loadId, userId]
  );
  if (existing) throw new Error('You have already bid on this load');

  return db.query(
    `INSERT INTO load_bids (load_id, bidder_user_id, amount) VALUES ($1, $2, $3)`,
    [loadId, userId, amount]
  );
}
```

3. Verify database rejects duplicate via constraint violation
