## ISSUE
Add RBAC to catalysts.ts. Procedures getStats, getProfile, getInvoices and all CRUD operations are missing requireAccess guards, allowing cross-company catalyst access.

## AFFECTED USER TYPES
- Catalyst

## FILE(S) TO MODIFY
- frontend/server/routers/catalysts.ts (all procedures)

## REQUIRED CHANGE

**Pattern to apply to all procedures:**
```typescript
// Before: No guards
async function getProfile(catalystId: string) {
  return db.query(`SELECT * FROM users WHERE id = $1`, [catalystId]);
}

// After: Add company isolation
async function getProfile(catalystId: string, req: Request) {
  await requireAccess(req, 'catalyst', targetCompanyId);
  const catalyst = await db.query(
    `SELECT u.* FROM users u WHERE u.id = $1 AND u.company_id = $2`,
    [catalystId, req.user.companyId]
  );
  if (!catalyst) throw new Error('Catalyst not found or access denied');
  return catalyst;
}
```

Apply same guard logic to: getStats, getProfile, getInvoices, updateProfile, deleteCatalyst.

## VERIFICATION
1. Login as Catalyst from Company A
2. Try to access Company B catalyst profile → access denied
3. Access own catalyst profile from Company A → success
4. Verify all CRUD operations enforce company boundary
